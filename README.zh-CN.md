# MedicineBoxNotes Android · 家庭药箱与病历本

一个原生、离线优先的 Android 应用，把分散在不同医院、抽屉里的纸质病历、处方、药盒标签与服药计划，汇聚成一本可搜索、留在设备本地的家庭健康簿 —— 配以有出处、保护隐私的本地 AI，你的数据从不上云。

**中文** | [English](./README.md) · [产品与技术方案](./doc/方案设计文档.md)

## 为什么做这个项目

家庭健康这件事，至今仍停留在「纸质时代」。病历是一叠散落的纸，分散在不同医院和抽屉里；家里的药箱是个黑盒；而最需要照顾的人，往往最容易忘掉一次药。MedicineBoxNotes 正是为解决这几件日常烦恼而生：

- **病历分散。** 纸质病历容易丢、更难拼起来——一个人的就诊史可能散落在好几家医院，从没有汇集在一处。
- **库存不清。** 买药回家才发现同款在柜子深处还躺着一盒没拆封。清楚地掌握家里到底备了什么药，能避免重复购买。
- **忘记服药。** 慢性病药只有按时吃才有效。一张「今日清单」让每一天该吃的药始终在眼前。
- **没法向自己的病历提问。** 「去年那场咳嗽我们吃过什么药？」这种问题应该几秒钟就能答出来，而不是翻箱倒柜找档案。

于是「药箱」成了一个安静、统一的地方：一眼看到家里备了多少药、用相机 + OCR + AI 随时翻看药品说明书，还能逐项勾掉今天要吃的药。

病历和处方是高度私人的内容，因此应用不会把它们上传到云端再去回答你的问题。一个本地大型语言模型（Gemma 4 E2B，经 LiteRT-LM 运行）全程在设备上完成自然语言问答——你在保有全部数据所有权的同时，依然能基于自己的病历得到快速、可对话的回答。即便设备上没有模型，内置的规则引擎也能让所有功能照常可用。

## 功能特性

五个标签页——首页、病历、药箱、查询、设置——全部使用 Jetpack Compose 构建。

**首页 / 今日**
- 双语日期与时段问候。
- 今日服药清单，由每款药的服药时间与用药计划自动生成；点击勾选一次服药（写入幂等的服药日志）。
- 距离最近一次复诊的倒计时。
- 库存预警（stock ≤ 5）与家庭成员快捷入口。

**病历**
- 家庭成员、病历、处方、图片附件的完整增删改查，基于 Room。
- 病历按就诊日期与年份分组，带成员筛选 Chip。
- CameraX 拍照与系统 Photo Picker 选图；私有原图 + 缩略图存储，配 ML Kit bundled OCR（先中文、后拉丁文回退）。
- 逐字段 OCR 智能填充——只填空字段，绝不覆盖你已输入的内容。
- 每行处方可「加入药箱」，并把来源（医院、就诊日期、成员）带入药品。
- 病历 A4 PDF 导出，经系统分享面板发出。
- 复诊提醒：每个复诊日期触发两次本地闹钟——前一天与当天，均为 09:00——重启或时区变更后自动重建，点击直达对应病历。

**药箱**
- 一份完整的家庭常备药清单，带库存徽标、AI 摘要与扫描数量。
- 扫描药盒正面 / 背面 / 侧面 / 说明书；按药品合并 OCR 文本，且不会把扫描类型名称误填进药品名。
- AI 智能整理把原始 OCR 转成结构化的名称 / 用量 / 频次 / 疗程——但只有你点击按钮时才执行，应用绝不自动写入 AI 字段。
- 手动编辑库存与用药计划（服药时间、起止日期、是否启用）。库存估算由「频次 × 疗程」得出。
- 库存预警（stock ≤ 5）。

**查询（端侧 AI）**
- 用自然语言向你的病历、附件 OCR/AI 文本与药箱提问，由设备上的 Gemma 4 E2B 作答。
- 答案以流式呈现，带处理动画、有据可依标识与可点击的来源引用。
- 每个 AI 入口都有风险确认门；回答过程可随时取消。
- 天然防幻觉：七条系统提示硬规则、全可选字段的 JSON 输出、清洗链，以及一道严格的安全门——任何没有来源、又不是诚实承认「没有证据 / 不知道」的回答都会被拒。

**设置**
- 应用内切换七种语言：默认英语，以及简体中文、日语、法语、德语、西班牙语、韩语。
- 家庭成员管理。
- 一键下载 Gemma 模型（断点续传、SHA-256 校验），支持开始 / 暂停 / 继续 / 删除。
- 加密备份导出与导入。

**贯穿全局**
- 离线优先：业务数据、图片、OCR 与 AI 推理默认都留在设备本地。
- 通过系统 SAF 的 AES-256-GCM 加密备份（`.mbn`），采用 PBKDF2 密钥派生（18 万次迭代）、SHA-256 文件清单与 UUID 合并导入。
- 暖纸感 Compose 设计系统：暖米色背景、白色 26dp 卡片、砖红主色、专属 AI 点缀色，并强制浅色主题。

## 架构设计

一个轻量的 Gradle 多模块工程——无 DI 框架、无图片加载库，由单一的 `MainViewModel` 把 Room 的 `Flow` 聚合成 `StateFlow`。

```text
app/                页面、导航、相机、提醒、PDF、备份、模型下载
core-model/         领域模型与库存/服药规则（纯 Kotlin/JVM）
core-database/      Room 实体、DAO、事务与 Repository
core-designsystem/  设计 Token 与可复用 Compose 组件
core-ai/            ML Kit OCR、规则 AI、Gemma LiteRT-LM 客户端
```

几处值得说明的设计取舍：

- **软锚定的药箱。** 药品与服药日志对病历、处方、成员都**不设外键**，仅用软锚定字段关联。删除病历或成员只是「摘除」药箱里的药品，而不是删除它——你手动维护的库存和服药历史不会因误删而消失。
- **抽象 + 规则回退。** 同一个 `MedicalAiClient` 接口有两个实现：始终可用的规则引擎，以及模型就绪后启用的 Gemma 客户端；任何出错或触发安全门时都透明地回退到规则引擎。应用绝不会因为模型缺失或失败而不可用。
- **GPU → CPU 引擎回退。** Gemma 引擎优先在 GPU 初始化，设备不支持时自动回退到 CPU，使同一套代码可在不同 arm64 设备上运行。
- **分层防幻觉。** 提示词硬规则 → 全可选字段的 JSON 输出 + 清洗 → 一道问答安全门：任何没有来源、又不是显式拒答的回答都会抛错。引用中内部的 UUID 标记在到达界面前会被剥离。
- **可断点续传、可校验的模型下载。** HTTP Range 续传由 `If-Range` ETag 校验把关，要求 6 GB 可用空间、≥2 GB 文件下限，并在下载完成后用固定 SHA-256 摘要复核。

## 技术栈

Kotlin 2.2.10 · JDK 17 · AGP 8.13.0 · Gradle 8.13。Jetpack Compose（BOM 2025.08.01）+ Material 3、Navigation Compose 2.9.3、Lifecycle 2.9.2、Room 2.8.4、CameraX 1.4.2、ML Kit 文本识别 16.0.1（拉丁文 + 中文）、LiteRT-LM 0.13.0、OkHttp 4.12.0、kotlinx-coroutines 1.10.2、kotlinx-serialization 1.9.0、DataStore 1.1.7、WorkManager 2.10.3。最低 Android 8.0（API 26）；编译/目标 SDK 36。Gemma 4 E2B（约 2.6 GB，`.litertlm`）来自 `huggingface.co/litert-community/gemma-4-E2B-it-litert-lm`。

## 构建与安装

在 `local.properties` 中配置 Android SDK 36（`sdk.dir=...`），然后：

```bash
./gradlew :app:assembleDebug
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

运行完整验证：

```bash
./gradlew :core-model:test :core-ai:testDebugUnitTest :app:lintDebug :app:assembleDebug :app:assembleRelease
```

如果工作区位于会生成 AppleDouble 文件的外置磁盘，工程会把每个模块的构建产物定向到 `/private/tmp/medicine-box-notes-build/${project.name}`，因此 Debug APK 位于：

```text
/private/tmp/medicine-box-notes-build/app/outputs/apk/debug/app-debug.apk
```

Release 产物经过 R8 压缩，但在你提供签名配置前默认未签名。Gemma 在运行时下载，不随 APK 打包；无模型或设备不支持时，应用自动保持规则模式，所有功能照常可用。

## 当前边界

- **本版本无账号、无云数据库、无跨端同步**——所有数据只存在单台设备上。
- **无逐次服药闹钟。** 今日服药是首页的 in-app 清单；只有复诊提醒会发出系统通知。
- **无药品效期追踪。** 药箱追踪库存与库存预警，暂不追踪保质期。
- **本地化为部分覆盖。** 仅英语完整；其余六种语言缺失的键会回退到英语，且复诊通知与 PDF 内容无论选择哪种语言都含硬编码中文。
- **AI/OCR 结果只是整理与检索的辅助，不能替代医生诊断**——使用前请务必核对原始材料。每个页面为此都保留了原始图片与 OCR 原文。
